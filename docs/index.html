Port Register

Zero-conflict port assignments for autonomous agents.

When multiple AI subagents spin up local dev servers, conflict is inevitable. Port Register provides a centralized, mandatory HTTP registry at localhost:4444 so agents can discover, claim, and release network resources safely.

View on GitHub

Conflict Detection

Scans both active Node registry memory and native OS properties (`netstat`, `tasklist`) to ensure absolutely zero bounds conflicts before a port is issued.

Dynamic Allocation

Agents can query `/api/suggest` bounds to automatically secure a randomized, free port without trial-and-error fallback loops.

Beautiful Dashboard

Monitor all allocations and system processes in real-time through the built-in dark-themed dashboard UI. See what every agent is doing cleanly.

Standard Integration

The mandatory bash invocation loop for agent scripts:

PORT=3000
BASE="http://localhost:4444/api"

STATUS=$(curl -s "$BASE/ports/check/$PORT")
AVAILABLE=$(echo $STATUS | python -c "import sys,json; print(json.load(sys.stdin)['available'])")

if [ "$AVAILABLE" = "True" ]; then
  curl -s -X POST "$BASE/ports/register" \
    -H "Content-Type: application/json" \
    -d "{\"port\":$PORT,\"agent\":\"Web-Agent\",\"reason\":\"React Server\"}"
else
  # Retrieve a free port suggestion
  FREE_PORT=$(curl -s "$BASE/suggest" | python -c "...\n")
  # Use FREE_PORT instead
fi
